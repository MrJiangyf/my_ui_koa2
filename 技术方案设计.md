1. 技术方案设计

   1. 架构设计：
      * 分层（api -> router -> controller -> cache -> server ->  db）
      * 意义：代码更加清晰

   2. 页面（模板，路由）和 API 设计

   3. 数据模型设计：

      * 4张表：blogs、users、userRelations、blogRelations

      * 各个表之间的关系（外键）

      * 三大范式：

        * 属性的原子性：每一列都不可再拆解

        * 记录的唯一性：有唯一标识（主键），其他属性都依赖于主键

        * 字段的沉余性：不存在数据沉余 和 传递依赖

        * 数据规范严谨，不易出错

        * 占用空间更小

        * 访问速度更快
